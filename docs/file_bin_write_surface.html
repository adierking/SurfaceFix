file_bin_write_surface(file, id)

Description

Writes a surface to an already opened binary file. This function is provided because GM's binary file functions only let you write one byte at a time, and writing surfaces byte-by-byte in a loop can be rather slow. The format of the data that is written is the same as the df_binary format used by surface_write().

This could be useful for saving surfaces along with your saved games.

Arguments

This function requires that you pass in 2 arguments:
  1. file - Real. The ID of the binary file to write the surface to. It must have been opened for write access.
  2. id - Real. The ID of the surface to write to the file. It must exist.

Return Value

Real. This function returns 1 (true) if it succeeds, or 0 (false) if it fails.

Example

// In this example, the "surface" variable holds the ID of a surface that was created earlier.

// Open a binary file for writing and clear it
file = file_bin_open("surface.dat", 1);
file_bin_rewrite(file);

// Write the surface to it
file_bin_write_surface(file, surface);

// Close the file
file_bin_close(file);
